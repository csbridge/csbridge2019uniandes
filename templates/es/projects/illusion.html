Ilusión óptica

Basado en una idea de Steve Cooper. Escrito por Chris Piech.

Folletos: Referencia Gráfica
Ejemplo solucionado: String Art Solution

Hay veces en las que los patrones repetitivos pueden hacer que nuestros ojos vean cosas que no están allí. Mira la imagen de abajo. ¿Puedes ver pequeños cuadrados grises en las intersecciones de las líneas blancas? Escribe un Programa EsGraphics que cree la ilusión óptica:

Constantes

Observa que el código de inicio para este proyecto tiene las líneas:

private static final int TAMANO = 100;
private static final int BRECHA = 10;
que se escriben fuera del método de ejecución (como el generador de números aleatorios). Estas líneas definen constantes: variables cuyos valores nunca cambian y como se definen fuera de un método, se puede obtener su valor desde cualquier parte del programa. Aunque se podría escribir esta ilusión óptica utilizando los números 100 y 10 directamente, usar constantes hace que su que tu programa sea mucho más fácil de leer.

Doble ciclo for

Este problema requiere usar un ciclo for dentro de un ciclo for. Recuerda utilizar una variable de iteración distinta para el ciclo interno. Por ejemplo, si tu ciclo externo es de la forma

for(int i = 0; i < 100; i++)
Entonces el ciclo interno debe iterar a través de una variable distinta de i.

Extensión
