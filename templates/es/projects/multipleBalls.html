Escrito por Tyler Conklin

Folletos: Referencia de generador aleatorio
Referencia Gráfica

Este es un programa extra! Está destinado a ser un reto.

Escribe un programa que anime dos bolas diferentes rebotando alrededor del lienzo. Si una bola golpea una pared, debería rebotar en la dirección opuesta. Las dos bolas también pueden rebotar una de la otra. Un ejemplo se muestra en el siguiente gif.

En esta carrera, el ancho de cada bola es de 50 píxeles, pero siéntase libre de usar los números que desee. Tenga en cuenta que al crear las bolas es importante darles diferentes velocidades aleatorias y posiciones X, Y iniciales. Consulte la referencia del generador aleatorio vinculado anteriormente para obtener ayuda para generar valores aleatorios.

Cuando los científicos informáticos abordan un gran problema, es útil dividirlo en pasos más pequeños. Una forma de resolver este problema sería:

  • Para comenzar, agregue dos bolas a la pantalla con diferentes posiciones aleatorias.
  • Consigue que las bolas se muevan con velocidades aleatorias.
  • Consigue las bolas para empezar a rebotar en las paredes correctamente. Recuerde que puede obtener la posición X de una bola GOval llamando a ball.obtenerX () (lo mismo ocurre con la posición Y).
  • Haz que las bolas reboten entre sí. Para esta parte, el método obtenerElementoEn, que se muestra a continuación, será útil.
